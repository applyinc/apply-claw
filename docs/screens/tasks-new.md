# Tasks New

## 0. 概要

- 機能名: タスク新規作成
- 画面 URL: `/tasks/new`
- 対象ユーザー: ワークスペース利用者
- 概要（50字程度）: cron ジョブを新規登録する専用画面
- 優先度: Must
- 関連リンク（Figma / チケット / PR）:

## 1. 目的（WHY）

### 課題

- 一覧画面と作成フォームが同居すると、ジョブ管理と登録作業の文脈が混ざる。

### 目標

- ユーザーが新規作成時だけ登録に集中できる画面を提供する。

## 2. できるようにしたい体験（ユーザーストーリー）

- ユーザーは `/tasks` から専用画面へ遷移し、必要項目を入力してジョブを登録できる。
- ユーザーは登録完了後に一覧へ戻り、新規ジョブを確認できる。

## 3. 今回の範囲

### Must（必須事項）

- 新規ジョブ作成フォームの表示
- 作成保存
- キャンセルして `/tasks` に戻る動線

### Won't（今回はやらない）

- 編集機能の統合
- 下書き保存

## 4. 対象画面・操作（WHAT）

### 表示要素

- 画面タイトル `New Job`
- 一覧へ戻るボタン
- ジョブ作成フォーム

### 操作仕様

#### 新規ジョブを登録する

- ユーザーは名前、説明、プロンプト、スケジュール、実行先、enabled を入力できる。
- 保存に成功すると `/tasks` に遷移し、新しいジョブを一覧で確認できる。

#### 作成をキャンセルする

- ユーザーは `Cancel` または `Back` から `/tasks` に戻れる。
- 戻ると一覧画面が表示される。
