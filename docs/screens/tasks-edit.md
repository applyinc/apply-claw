# Tasks Edit

## 0. 概要

- 機能名: タスク編集
- 画面 URL: `/tasks/[jobId]/edit`
- 対象ユーザー: ワークスペース利用者
- 概要（50字程度）: 既存 cron ジョブを更新する専用画面
- 優先度: Must
- 関連リンク（Figma / チケット / PR）:

## 1. 目的（WHY）

### 課題

- 一覧画面に編集フォームを同居させると、閲覧と編集の文脈が混ざる。

### 目標

- ユーザーが編集時だけ設定変更に集中できる画面を提供する。

## 2. できるようにしたい体験（ユーザーストーリー）

- ユーザーは `/tasks` の編集ボタンから専用画面へ遷移し、既存ジョブを更新できる。
- ユーザーは保存完了後に一覧へ戻り、更新内容を確認できる。

## 3. 今回の範囲

### Must（必須事項）

- 既存ジョブ編集フォームの表示
- 更新保存
- キャンセルして `/tasks` に戻る動線

### Won't（今回はやらない）

- ジョブ複製
- 実行履歴の詳細表示

## 4. 対象画面・操作（WHAT）

### 表示要素

- 画面タイトル `Edit Job`
- 一覧へ戻るボタン
- ジョブ編集フォーム

### 操作仕様

#### 既存ジョブを編集する

- ユーザーは `/tasks` から編集ボタンを押して `/tasks/[jobId]/edit` へ遷移できる。
- 遷移に成功すると、既存ジョブの設定値が入ったフォームを確認できる。
- 保存に成功すると `/tasks` に遷移し、更新後のジョブを一覧で確認できる。

#### 編集をキャンセルする

- ユーザーは `Cancel` または `Back` から `/tasks` に戻れる。
- 戻ると一覧画面が表示される。

#### 存在しないジョブへアクセスする

- ユーザーが存在しない `jobId` でアクセスした場合、ジョブ未検出メッセージを確認できる。
- `Back to Jobs` を押すと `/tasks` に戻れる。
