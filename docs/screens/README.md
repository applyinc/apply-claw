# 画面仕様書

このディレクトリでは、画面ごとの仕様を Markdown で管理する。

## ルール

- 1 画面 1 ファイル。
- ファイル名は原則としてルートに対応させる。
- 新規画面追加時は `docs/screens/_templates/screen-spec-template.md` をコピーして作成する。
- API 仕様が多い場合は同ファイル内の「8. API 契約」を更新し、必要なら `docs/api/` に詳細を分離する。
- 「4. 対象画面・操作（WHAT）」の「操作仕様」は、ユーザー目線のユースケース形式で記述する。
- 「操作仕様」は `####` 見出しで操作単位に分割し、操作結果（成功/失敗/権限制御）を書く。

## 一覧

- `tasks.md`: `/tasks` のタスク管理画面仕様
- `tasks-new.md`: `/tasks/new` のタスク新規作成画面仕様
- `tasks-edit.md`: `/tasks/[jobId]/edit` のタスク編集画面仕様
- `workspace.md`: `/` のワークスペース画面仕様
