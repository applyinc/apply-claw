# Tasks

## 0. 概要

- 機能名: タスク管理
- 画面 URL: `/tasks`
- 対象ユーザー: ワークスペース利用者
- 概要（50字程度）: エージェント向けの定期実行ジョブを一覧・遷移操作で管理する画面
- 優先度: Must
- 関連リンク（Figma / チケット / PR）:

## 1. 目的（WHY）

### 課題

- 定期実行ジョブの作成や停止をコード編集なしで操作できない。
- ジョブの状態や前回実行時刻を一覧で確認しづらい。

### 目標

- ユーザーが `/tasks` で cron ジョブを一元管理できる。
- 作成・編集・有効無効切り替え・削除を画面上で完結できる。

## 2. できるようにしたい体験（ユーザーストーリー）

- ユーザーはジョブ一覧を開き、現在の設定と状態をひと目で把握できる。
- ユーザーは新規作成画面へ遷移し、定期実行条件とエージェントへのプロンプトを保存できる。
- ユーザーは既存タスクの設定を編集し、不要になったタスクを停止または削除できる。

## 3. 今回の範囲

### Must（必須事項）

- タスク一覧の表示
- 新規作成画面への導線表示
- タスク編集画面への導線表示
- タスクの有効無効切り替え
- タスクの削除

### Won't（今回はやらない）

- タスクの検索・絞り込み
- 実行履歴の詳細表示
- 並び順変更

## 4. 対象画面・操作（WHAT）

### 表示要素

- 画面タイトル `Tasks`
- 新規作成ボタン
- タスク一覧カード
- 空状態メッセージ
- サマリーカード

### 操作仕様

#### タスク一覧を確認する

- ユーザーは `/api/cron/jobs` から取得したタスク一覧を確認できる。
- 取得に成功すると、各タスクの名前、スケジュール、状態、前回実行時刻、説明が表示される。

#### タスクを新規作成する

- ユーザーは `New Job` を押して `/tasks/new` へ遷移できる。
- 遷移に成功すると、新規作成専用画面で名前、説明、プロンプト、スケジュール、実行先、enabled を入力できる。

#### タスク編集画面へ遷移する

- ユーザーは既存タスクの編集ボタンを押して `/tasks/[jobId]/edit` へ遷移できる。
- 遷移に成功すると、既存ジョブの設定値が入った編集画面を確認できる。

#### タスクを有効化・無効化する

- ユーザーは再生または停止アイコンを押して、タスクの enabled 状態を切り替えできる。
- 更新に成功すると、一覧が再取得され、状態ラベルとアイコン表示が切り替わる。

#### タスクを削除する

- ユーザーは削除ボタンを押すと確認ダイアログを経由してタスクを削除できる。
- 削除に成功すると、一覧が再取得され、対象タスクが表示から消える。

## 8. API 契約

- 画面が利用する API: `GET /api/cron/jobs`, `POST /api/cron/jobs`, `PUT /api/cron/jobs`, `DELETE /api/cron/jobs`
- リクエスト: JSON でジョブ設定または `id` を送信する
- レスポンス: ジョブ一覧または更新結果を返す
- エラー時挙動: 現状は専用エラー表示なし。保存・更新失敗時は一覧更新が行われない

## 9. バリデーション・エラーハンドリング

- 入力バリデーション: 名前が空の場合は保存ボタンを押せない
- エラーメッセージ: 現状は画面内メッセージ未実装
- 失敗時の復帰方法: ユーザーは入力内容を維持したまま再実行できる

## 10. 非機能要件
